## 范围与隔离边界

SQLite 继续作为唯一机器状态源；本文仅是从实现与测试结果生成的可读证据，不参与状态判定。

## 本轮补强

- Agent 子进程支持运行中取消轮询，并把 `cancelled` 独立持久化到 `agent_runs` 与事件表。
- Agent 子进程超时持久化为 `timed_out`，不再与普通失败混淆。
- Job 生命周期把取消作为终态，不生成失败案例，也不进入重试。
- 超时和普通失败按照 `max_attempts` 进入 `retry_wait`，耗尽次数后进入 `failed`。
- 重试允许同一 Job 产生新的 lease；旧库启动时迁移遗留的单 Job 唯一 lease 约束。
- 独立复核从 SQLite 读取执行者身份，拒绝同 Agent 自审。
- 独立复核同时检查执行结果与复核结果；执行失败时，即使复核 Agent 输出 `PASS`，最终验证仍失败。
- 复核证据包含执行与复核 Agent Run ID、状态、退出码、输出及规范化 SHA-256 摘要。

本轮新增的第五阶段生命周期测试覆盖：

1. 超时被持久化，并在最大尝试次数内重试，耗尽后失败；
2. 运行中取消会终止 Agent，且永不重试；
3. 自审被拒绝，另一 Agent 的复核身份与结论写入 SQLite；
4. 复核 Agent 的 `PASS` 不能把失败的执行结果变成成功。

## 阶段判定

第五阶段的真实隔离执行与既有跨 Agent 复核证据继续有效；本轮进一步补齐了失败、取消、超时和重试的机器生命周期闭环。Executor 仍只能报告结果，不能自行验证或完成 Goal；最终验证由独立复核路径写入，Goal 完成仍需后续决策链。
