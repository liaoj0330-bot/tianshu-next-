# TianShu P4 长期运行治理｜计划

## 目标

让天枢能够持续运行、可恢复、可取消、可并发控制，并在系统重启或执行异常后不猜测成功。

## 建设顺序

### 1. 持久任务队列

- Goal、Plan、Task、Run 与 Project 关联；
- 任务状态持久化；
- 幂等入队；
- 任务取消；
- 失败原因结构化记录。

### 2. Worker 租约

- Worker 领取任务时写入租约；
- 租约过期后进入恢复检查；
- 同一项目的互斥锁；
- 不允许重复领取同一任务。

### 3. 进程与资源治理

- Codex 子进程树跟踪；
- 超时和取消时回收完整进程树；
- 孤儿进程身份核验；
- 临时 Worktree 和产物清理；
- 日志脱敏和证据哈希。

### 4. 保守恢复

- 服务重启后对账；
- `running` 不直接当作成功；
- 中断任务进入 `recovery_required`；
- 部分产物进入待验收状态；
- 失败后有限重试，不无限循环。

### 5. 并发与优先级

- 项目级锁；
- 任务优先级；
- 依赖关系；
- 同一项目串行、不同项目受控并行；
- 高风险任务必须停在审批门。

## P4 通过标准

- 重启后任务状态不丢失；
- 同一任务不会被两个 Worker 同时执行；
- 超时能回收子进程树；
- 取消后不会留下可误认的成功状态；
- 部分产物可追踪、可清理、可重新验收；
- 失败可重试但有上限；
- 任一适配器断开，SQLite 机器真相仍然完整；
- Teacher PPT 和业务项目仍保持关闭，直到迁移门批准。

## P4 明确不做

- 不自动切换正式入口；
- 不开放业务项目无人值守写入；
- 不接入多个外部消息入口；
- 不把 Obsidian 当作运行时数据库；
- 不因一次成功运行就宣称长期稳定。
