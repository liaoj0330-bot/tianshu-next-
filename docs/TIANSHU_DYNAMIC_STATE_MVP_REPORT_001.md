## 已实现能力

1. `stable / current / future` 三层状态分开保存。
2. 当前状态使用 SQLite，不把 Markdown 当机器真相。
3. 每次输入先生成不可变更新提案，不直接覆盖当前状态。
4. 只有高置信度且来源为奈奈明确表达或已验证证据的当前信号，才能进入建议更新。
5. 临时推测、低置信度和无授权来源只能进入待确认候选。
6. 每轮最多提出 3 个问题；已问或已答的问题不重复询问。
7. 多个候选动作只选择一个下一步。
8. 奈奈可以接受、纠正或拒绝；纠正结果成为权威新快照。
9. 新快照成立后，旧快照转为历史，不覆盖删除。
10. 基于旧快照的并发提案不能覆盖更新后的状态。
11. 可生成一屏中文决策卡，隔离运行目录之外不写文件。
