# 天枢 Obsidian 读模型阶段证据 001

## 1. 阶段目标

本阶段建立一个由 SQLite 单向生成、可重复构建、可恢复的 Obsidian 知识工作台，让奈奈能够阅读和确认天枢状态，同时不把 Markdown 误设为机器状态真相源。

## 2. 三层职责

- AgentHub：奈奈的统一交互入口，负责接收上下文、组织智能体协作、呈现需要奈奈确认的事项。
- SQLite：唯一机器状态真相源，保存权威状态、候选项、决策、执行、验证和验收记录。
- Obsidian：奈奈可读、可确认的知识工作台，是 SQLite 的可再生读模型，不直接完成状态写入。

## 3. 已实现范围

新增 `src/product/obsidian-read-model.mjs`，从 SQLite 构建以下工作台骨架：

```text
00_从这里开始_天枢工作台.md
10_奈奈/00_奈奈模型.md
20_项目/00_项目总览.md
20_项目/项目_<名称>_<稳定标识>.md
30_生活与关系/00_生活与关系.md
40_知识与判断/00_知识与判断.md
40_知识与判断/01_外部建议审议.md
50_进化/00_进化总览.md
60_活动/00_活动记录.md
99_系统/00_系统说明.md
99_系统/01_生成清单.md
.tianshu-read-model.json
```

旧目录不再继续横向堆叠，而按职责收敛：

- `90_待处理`：收敛为 Today 中由 SQLite 查询得到的待确认事项。
- `98_历史归档`：收敛为 Activity 和 Evolution 的时间视图。
- `06_项目记忆层`：收敛到每个项目页的时间线。
- `07_资产索引层`：收敛到项目页及活动记录中的证据索引。

新增命令入口：

```powershell
npm run obsidian-read-model -- --state <state.sqlite> --output <isolated-directory>
```

命令拒绝把 SQLite 状态文件放在输出目录内部，避免生成或清理流程影响权威状态。

## 4. 已验证不变量

1. 生成前后 SQLite 全表快照一致，生成器不写机器状态。
2. 同一 SQLite 状态重复生成，所有 Markdown 和清单字节级一致。
3. 删除生成页后可以等价再生。
4. 人工修改现有生成页后，再次生成会由 SQLite 权威状态恢复。
5. 未登记在生成清单中的奈奈个人文件不会被删除或覆盖。
6. 旧版生成页只有在仍与上一版哈希一致时才会被安全清理；已人工修改的旧页会保留。
7. 首页中的所有 Obsidian 内链均可解析到生成文件。
8. 清单记录每个生成文件的 SHA-256 与字节数，可独立核验。

## 6. 阶段边界

本阶段没有：

- 读取或改造奈奈当前正在使用的外部 Obsidian 仓库；
- 允许 Markdown 直接修改 SQLite；
- 把 Obsidian 当作新的操作入口或审批入口；
- 实现非受控的外部报告自动入库；
- 用执行者的输出代替独立验证或奈奈最终验收。

## 7. 阶段结论与下一入口

Obsidian 读模型阶段门禁通过。下一阶段应把 Today 固定为“需要奈奈注意和决策的唯一驾驶舱”，并让 AgentHub 的所有输入、澄清、确认和执行申请进入同一网关；项目、知识、生活、进化和活动仍各自保留为稳定工作区，不再把所有内容堆到 Today。
